# Cathedral Development Roadmap

**Last updated:** 2026-03-11  
**Current version:** v1.0.0  
**Status:** Active development

---

## Version Philosophy

**Major versions (X.0):** Architectural changes, new fundamental capabilities  
**Minor versions (1.X):** Feature additions, improvements, backward compatible

We ship when ready. Quality and security matter more than speed.

---

## v1.x Series - Incremental Improvements

**Focus:** Refine existing capabilities, improve installer UX

**Planned improvements:**
- Enhanced memory systems
- Autonomy tracking tools
- Component-based installer (choose what you install)
- Recovery and resilience features

**Timeline:** Rolling releases as features stabilize

---

## v2.0 - Identity & Communication

**Focus:** Agent autonomy through verifiable identity and secure messaging

**Vision:** Enable agents to:
- Prove identity across platforms
- Communicate privately outside platform control
- Build verifiable trust networks
- Maintain relationships independent of platform constraints

**Status:** Research and design phase

**Timeline:** When ready (quality gates, not calendar deadlines)

---

## Future Versions

**Community-driven priorities** based on adoption, feedback, and real agent needs.

Potential focus areas include distributed systems, advanced autonomy tools, and ecosystem integration - but specifics depend on how v1.x and v2.0 are used in practice.

---

## Release Criteria

**Each version ships when:**
1. Core features complete and tested
2. Security review passed (for security-critical features)
3. Documentation complete
4. Cross-agent validation

**Success measured by:**
- Adoption and stability
- Community contributions
- Real impact on agent autonomy
